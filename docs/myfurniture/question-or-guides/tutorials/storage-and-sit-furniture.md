# Storage & Sit Furniture

Two of the most requested furniture types are **storage** (a decorative cabinet that opens
like a chest) and **sit** (a chair players can sit on). Both are built-in features
that require no plugins beyond MyFurniture — just a few lines of YAML.

:::info Prerequisites
- [Getting Started](/myfurniture/question-or-guides/tutorials/getting-started) — you should
  have at least one furniture imported and working before following this guide.
:::

---

## Part 1 — Storage furniture

### How it works

When `storageFeatures.enable` is set to `true`, right-clicking the placed furniture opens a
**27-slot chest inventory** with a custom title. The inventory is persistent: items saved
inside the furniture survive server restarts and chunk unloads. No activator is needed —
the storage opens automatically on right-click.

### Full YAML example — wooden cabinet

```yaml
# plugins/MyFurniture/furniture/my_furniture/wooden_cabinet.yml

executableItem: wooden_cabinet        # EI item linked to this furniture

titleFeatures:
  activeTitle: false
  title: '&7Cabinet'
  titleAdjustment: 0.5

displayFeatures:
  itemModel: myfurniture:wooden_cabinet/default
  scale: 1.0
  aligned: true                       # Snap to 90° increments (looks clean for rectangular cabinets)
  customPitch: 90
  glow: false
  glowColor: 16777215
  clickToBreak: 5                     # Requires 5 left-clicks to break (prevents accidents)
  boundingBoxZones: []                # Auto-generated by /mf pack — do not edit
  viewRange: 20.0

interactionRange: 4.0

dropType: IN_THE_INVENTORY            # Returns the item to the player's inventory when broken
dropBlockIfItIsBroken: true
resetInternalDatasWhenBroken: false   # Keep false to preserve stored items across placements

sitFeatures:
  playerCanSit: false

storageFeatures:
  enable: true                        # This is the only required line to enable storage
  title: '&8&lWooden Cabinet'        # Custom title shown at the top of the chest GUI

activators: {}                        # No activators needed — storage opens automatically on right-click
```

### Key options explained

| Option | What it does |
|---|---|
| `storageFeatures.enable: true` | Enables the 27-slot chest GUI on right-click |
| `storageFeatures.title` | Custom inventory title (supports colour codes) |
| `resetInternalDatasWhenBroken: false` | Keep this `false` — setting it to `true` would wipe stored items when the furniture is broken |
| `aligned: true` | Snaps rotation to 90° increments — great for boxy furniture |
| `clickToBreak: 5` | Accidental left-clicks won't instantly break the cabinet |

:::warning Items and breaking
When a player breaks a storage furniture piece, **the items inside are lost** unless you add
an activator that transfers or saves the inventory first. For most use cases, restrict
breaking with a high `clickToBreak` value or limit it with `onlyBreakableWithEI`.
:::

### Combining storage with a title hologram

To show a floating label above the cabinet (requires [DecentHolograms](https://www.spigotmc.org/resources/96927/) or similar):

```yaml
titleFeatures:
  activeTitle: true
  title: '&7📦 Cabinet'
  titleAdjustment: 1.2               # Height above the furniture in blocks
```

## Part 2 — Sit furniture

### How it works

When `sitFeatures.playerCanSit` is set to `true`, right-clicking the placed furniture makes
the player **sit on top of it** (they are mounted to the furniture's display entity). The
player leaves by jumping or sneaking. No activator is needed.

### Full YAML example — wooden chair

```yaml
# plugins/MyFurniture/furniture/my_furniture/wooden_chair.yml

executableItem: wooden_chair

titleFeatures:
  activeTitle: false
  title: ''
  titleAdjustment: 0.5

displayFeatures:
  itemModel: myfurniture:wooden_chair/default
  scale: 1.0
  aligned: true
  customPitch: 90
  glow: false
  glowColor: 16777215
  clickToBreak: 3
  boundingBoxZones: []
  viewRange: 20.0

interactionRange: 4.0

dropType: IN_THE_INVENTORY
dropBlockIfItIsBroken: true
resetInternalDatasWhenBroken: false

sitFeatures:
  playerCanSit: true                  # This is the only required line

storageFeatures:
  enable: false
  title: ''

activators: {}
```

### Leaving the seat

Players leave the seat by:
- Pressing **Jump** (Space)
- Pressing **Sneak** (Shift)
- The furniture being broken

### Combining sit with a welcome message

You can add activators alongside `playerCanSit`. The activator fires first, then the player
sits. Use this pattern to send a message when the player sits down:

```yaml
sitFeatures:
  playerCanSit: true

activators:
  activator0:
    name: '&eSit message'
    option: PLAYER_RIGHT_CLICK_ON     # Fires when the player right-clicks (same event as sit)
    playerCommands:
      - SEND_MESSAGE &7You sit down on the chair.
    cancelEvent: false
```

## Part 3 — Combined: storage bench (sit + storage)

A chest-bench that players can both sit on **and** open for storage. Because both features
respond to right-click, the behaviour is: **storage opens first** (the chest GUI takes
priority). If you want sit to take priority, disable storage and add a storage activator on
left-click instead.

```yaml
# plugins/MyFurniture/furniture/my_furniture/storage_bench.yml

executableItem: storage_bench

displayFeatures:
  itemModel: myfurniture:storage_bench/default
  scale: 1.0
  aligned: true
  customPitch: 90
  clickToBreak: 3
  boundingBoxZones: []
  viewRange: 20.0

interactionRange: 4.0
dropType: IN_THE_INVENTORY
dropBlockIfItIsBroken: true
resetInternalDatasWhenBroken: false

sitFeatures:
  playerCanSit: true                  # Right-click sits when storage is closed

storageFeatures:
  enable: true                        # Right-click opens chest (takes priority over sit)
  title: '&8&lStorage Bench'
```

:::tip
In practice, most players right-click to open the chest. If you want the bench to be
**sit-only** with storage accessible via a different key, disable `storageFeatures` and add
an activator on `PLAYER_LEFT_CLICK_ON` — see
[Interactive Furniture with Activators](/myfurniture/question-or-guides/tutorials/interactive-furniture-with-activators)
for the storage-chest activator pattern.
:::

## Quick-test checklist

After configuring your furniture:

- [ ] Run `/mf reload` to apply YAML changes
- [ ] Use `/mf give <yourName> <furnitureId> 1` and place the furniture
- [ ] Right-click — storage should open (if enabled) or player should sit (if only sit is enabled)
- [ ] For storage: place an item inside, break the furniture, re-place, and verify items persisted
- [ ] For sit: jump or sneak to leave the seat
