## What Makes a Custom Block Plugin?

Custom block plugins fall into two camps:

1. **Behavior-first**: Define what a block *does* — triggers, conditions, commands, cooldowns. The block looks like a normal vanilla block (or a retextured one via a pack).
2. **Texture-first**: Add new block visuals into the game via a resource pack. Behavior is secondary, limited, or requires companion plugins.

ExecutableBlocks is firmly behavior-first. ItemsAdder, Nexo, and Nova are texture-first with varying behavior support.

## ExecutableBlocks — Interactive Blocks Without Limits

### What Is It?

[ExecutableBlocks](https://docs.ssomar.com/executableblocks/information-eb) is a YAML-driven plugin that lets you add **event-based abilities** to any Minecraft block — without modifying the block's visual appearance (though you can apply custom textures too).

It shares the same SCore engine as [ExecutableItems](https://docs.ssomar.com), meaning blocks can use the same 30+ plugin integrations, condition system, and command library as EI items.

### Core Strengths

#### 30+ Block Activators

ExecutableBlocks triggers abilities based on block-related events:

| Activator | When it fires |
|---|---|
| `PLAYER_RIGHT_CLICK_ON` | Player right-clicks the block |
| `PLAYER_LEFT_CLICK_ON` | Player left-clicks the block |
| `PLAYER_BREAK` | Player breaks the block |
| `PLAYER_PLACE` | Player places the block |
| `PLAYER_WALK_ON` | Player walks onto the block |
| `PLAYER_JUMP_ON` | Player jumps on the block |
| `PLAYER_FALL_ON` | Player lands on the block |
| `PLAYER_SNEAK_ON` | Player sneaks on the block |
| `ENTITY_WALK_ON` | Any entity walks on it (premium) |
| `LOOP` | Repeatedly, every N ticks (premium) |
| `EXPLOSION_HIT` | Hit by an explosion (premium) |
| And more... | |

This makes blocks interactive in ways vanilla Minecraft simply doesn't support.

#### Full Condition System

Every activator checks conditions before running: player permissions, health, level, WorldGuard regions, PlaceholderAPI values, random chance, time of day, and more. A pressure plate that only triggers for VIP players, a spawner that only works at night — all straightforward.

#### Block-Specific Commands

In addition to all the SCore player commands (also available in ExecutableItems), EB adds block-context commands:

- `EXPLODE` — break the block and spawn a primed TNT
- `MINEINCUBE radius:3` — mine a 3×3×3 cube of blocks around it
- `SETBLOCK {material}` — replace the block with another type
- `SETTEMPBLOCK {material} {ticks}` — temporarily replace and revert
- `VEINBREAKER` — break an entire connected vein of the same block type
- `FARMINCUBE` — apply farming actions to surrounding crops

#### Seamless ItemsAdder and Nexo Import

ExecutableBlocks can **import** your ItemsAdder or Nexo block configurations directly:

```
creationType: IMPORT_FROM_ITEMSADDER
```

This means you can keep your resource-pack assets in ItemsAdder and add interactive behavior on top with ExecutableBlocks — the best of both worlds.

### Use Cases

**Regenerating ore vein:**
```yaml
activators:
  activator0:
    option: PLAYER_BREAK
    blockCommands:
      - DELAY 30
      - SETBLOCK DIAMOND_ORE    # Respawn after 30 seconds
    playerCommands:
      - SEND_MESSAGE &7This vein respawns in 30 seconds.
```

**Pressure plate that charges entry fee:**
```yaml
activators:
  activator0:
    option: PLAYER_WALK_ON
    playerConditions:
      ifHasPermission:
        - arena.pass
    commands:
      - [CONSOLE] economy take %player% 100
      - [CONSOLE] teleport %player% arena_spawn
```

**Custom TNT block:**
```yaml
activators:
  activator0:
    option: PLAYER_BREAK
    cancelEvent: true
    blockCommands:
      - EXPLODE
```

### Ease of Configuration

Like ExecutableItems, each block lives in its own YAML file and there's a full in-game GUI (`/eb editor`). You can create, test, and reload blocks without restarting the server.

## ItemsAdder — The Texture-First Block Platform

### What Is It?

ItemsAdder treats custom blocks primarily as **resource-pack assets**: new block shapes, textures, and sounds bundled into a pack that's auto-delivered to players.

### Core Block Strengths

- **Custom block shapes** (using note block states or armor stand hacks depending on version)
- **Custom block sounds** and particle effects
- Furniture and decoration blocks
- A large ecosystem of pre-made block content packs

### Block Behavior in ItemsAdder

ItemsAdder's block *behavior* system is limited compared to ExecutableBlocks. Basic actions (on-break drops, custom crafting recipes for the block) are built in, but complex triggers, conditions, and command chains require scripting or a companion plugin.

**When to choose:** Your primary goal is custom visual blocks that fit a resource-pack workflow and you don't need complex interactive behavior.

---

## Nexo — Modern Resource-Pack Framework

### What Is It?

Nexo (the Oraxen successor) takes a developer-friendly approach to custom content, with a clean API and strong resource-pack support. Block types, models, and sounds are Nexo's core product.

### Block Behavior in Nexo

Nexo's interactive behavior system is growing but still lags behind ExecutableBlocks. Developer add-ons can hook into Nexo's events for more complex behavior.

**When to choose:** Developer-first servers that want a modern API to build on, with custom block textures as a priority.

---

## Nova — Block Framework for Developers

### What Is It?

Nova is a plugin *framework* for adding new blocks with custom behavior, targeted at plugin developers. Server admins generally don't configure Nova directly — they install Nova *addons* built by developers.

### Strengths and Limitations

- Very powerful block simulation (custom storage, machines, energy systems)
- High learning curve — primarily for developers, not admin configuration
- Requires a resource pack
- Smaller community and fewer ready-to-use addons compared to other options

**When to choose:** You are a developer building a custom tech/magic plugin and need deep block behavior simulation (machines, energy grids, custom containers).

## Real-World Use Cases

### "I want blocks that run commands when players step on them."

**ExecutableBlocks.** `PLAYER_WALK_ON` activator with any player commands.

### "I want a block that respawns 60 seconds after being mined."

**ExecutableBlocks.** `PLAYER_BREAK` → `DELAY 60` → `SETBLOCK {material}`.

### "I want custom-textured stone blocks that look like ancient ruins."

**ItemsAdder** or **Nexo.** These are designed for custom textures, custom block models, and themed content packs.

### "I need custom machine blocks with energy and item processing."

**Nova** (developer route) or a plugin like SlimefunFour.

### "I want a veinminer pickaxe that auto-mines a 3×3 area."

This is actually an item ability — use **ExecutableItems** with `MINEINCUBE` in a `PLAYER_BLOCK_BREAK` activator.

## Pairing ExecutableBlocks with ItemsAdder

The most powerful setup many servers use:

1. **ItemsAdder** manages the resource pack, providing custom block textures, models, and sounds
2. **ExecutableBlocks** adds interactive behavior on top, using the `creationType: IMPORT_FROM_ITEMSADDER` setting

This separates concerns cleanly: artists work in ItemsAdder, behavior is configured in ExecutableBlocks YAML.
